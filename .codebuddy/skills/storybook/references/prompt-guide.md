# 图片 Prompt 编写指南

## 核心原则：一致性优先

绘本最大的挑战是跨页面的视觉一致性。每次 API 调用是独立的，模型没有"记忆"。
必须通过以下策略保持一致性：

### 策略 1: 角色参考图（最关键）

在生成所有页面之前，**先生成一张角色参考图**（character sheet）。

### 策略 2: Prompt 中重复角色描述

即使有参考图，每一页的 prompt 中也要包含角色的关键视觉特征描述。这是双重保障。

**每页 prompt 的结构**：

```
[STYLE DIRECTIVE]
画风和色调的整体指令

[SCENE DESCRIPTION]
这一页的具体场景和构图

[CHARACTER DETAILS]
出场角色的关键外貌描述（从 characters.md 摘取）

[CONSISTENCY ANCHOR]
强调一致性的指令
```

### 策略 3: 风格锚定词

在每页 prompt 的开头和结尾都加上统一的风格锚定词，确保画风不漂移。

## Prompt 模板

### 封面 Prompt 模板

```markdown
【画风】儿童绘本插画，[画风]风格，[色调]色调，16:9 横版构图，2K 高清画质。

【场景】绘本封面设计。[封面构图和场景描述]。

【标题文字】画面[位置，如顶部/上方/中央]用[字体风格，如圆润可爱的手写体/卡通艺术字]写着「[绘本中文标题]」几个大字，字体颜色为[颜色]，与画面风格协调，采用中英双语。

【角色】
- [角色名]：[关键外貌描述，从 characters.md 摘取最识别性的 3-5 个特征]

【画质】专业儿童绘本插画品质，构图干净，画风统一为[画风]风格。
```

### 内页 Prompt 模板

```markdown
【画风】儿童绘本插画，[画风]风格，[色调]色调，16:9 横版构图。这是绘本的第 [N] 页，保持与其他页面一致的画风。

【场景】[场景时间和地点]。[镜头类型：远景/中景/近景/特写/俯视]。[详细的画面描述：角色在做什么、环境细节、光线、氛围]。

【角色】
- [角色名]：[完整外貌描述]。表情：[这一页的表情]。动作：[这一页的动作/姿态]。
- [角色名]：[完整外貌描述]。表情：[表情]。动作：[动作]。

【构图】[构图方式]。[视觉焦点说明]。如果画面中有招牌、书本封面、信件、标语等自然文字元素，用中文书写（如：招牌上写着「[中文内容]」）。

【故事文字】画面[底部/顶部]用[字体风格，如柔和的手写体/圆润卡通字体]写着「[这一页的故事正文，1-3句]」，字体颜色为[颜色]，字号适中清晰可读，与画面风格协调融为一体，采用中英双语。

【氛围】这一页的情绪是[情绪关键词]。色温：[暖/冷/中性]。光线：[光线描述]。

【一致性】保持角色外貌与设定完全一致：[角色名]必须有[最关键的 2-3 个识别特征]。画风始终保持[画风]风格，色调与全书统一。
```

## Prompt 编写注意事项

### DO（应该做的）

- **用中文写 prompt** —— 避免模型因英文语境而在画面中生成英文文字
- **具体描述** —— 「一只橘红色的小狐狸，戴着天蓝色围巾」而不是「一只可爱的狐狸」
- **每页重复角色特征** —— 不要假设模型记得之前的 prompt
- **指定画风和色调** —— 每页都明确写出
- **描述光线和氛围** —— 这决定了画面的情绪
- **指定构图和镜头** —— 远景、中景、近景、特写、俯视等
- **渲染故事文字** —— 在 prompt 中指定故事正文的文字内容、位置、字体风格和颜色，让模型直接在画面中渲染
- **图片中的文字使用中文** —— 除专有名词外，画面中出现的所有文字（如封面标题、场景中的招牌等）都应该是中文

### DON'T（不要做的）

- **不要用抽象描述** —— 「美丽的场景」没有信息量
- **不要省略角色描述** —— 即使有 --ref，prompt 中也要写
- **不要用英文写 prompt** —— 英文 prompt 会导致模型在画面中生成英文文字
- **不要要求太多元素** —— 一页聚焦一个核心场景，不要塞太多内容
- **不要忘记 aspect ratio** —— 绘本页面是 16:9 横版

## 关于文字渲染

NanoBananaPro 模型能够较好地处理中文文字渲染，但需要正确引导。

### 核心原则

1. **所有画面中的文字必须使用中文**（除专有名词外）
2. **必须明确写出具体的中文文字内容**，不能只说「写着标题」，要说「写着'闪亮的牙齿'五个中文大字」
3. **指定字体风格和颜色**，让模型知道文字的视觉效果

### 封面：直接渲染中文标题

封面必须让模型在画面中渲染中文标题文字，而不是「预留区域」。方法：
- 在 prompt 中明确指定：标题文字内容、位置、字体风格、颜色
- 示例：「画面顶部用圆润可爱的卡通字体写着'小狐狸找妈妈'六个中文大字，字体为白色带浅橙色描边」
- 标题文字应与画面构图融为一体，而不是后期叠加

### 内页：分两种情况

**故事正文区域** → 让模型直接在画面中渲染中文文字：
- 在 prompt 中明确写出这一页的故事正文内容（1-3 句，不要太长）
- 指定文字在画面中的位置（如「画面底部」「画面顶部」）
- 指定字体风格（如「柔和的手写体」「圆润的卡通字体」）和颜色
- 文字应与画面构图融为一体，自然美观
- 示例：「画面底部用柔和的手写体写着'小狐狸轻轻推开了门，屋子里暖洋洋的。'，字体为深棕色，与画面风格协调」

**场景中的自然文字**（招牌、书本、信件、标语等）→ 让模型直接渲染中文：
- 必须在 prompt 中写出具体的中文文字内容
- 示例：「面包店门口的招牌上用楷体写着'熊猫烘焙坊'四个中文字」
- 示例：「黑板上用粉笔写着'欢迎新同学'五个中文大字」

## 分辨率和尺寸

| 参数 | 值 | 说明 |
|------|------|------|
| 宽高比 | 16:9 | 横版绘本，适合屏幕阅读和打印 |
| 质量 | 2K | 2048px 长边，足够打印和高清显示 |
| 格式 | JPEG | 压缩格式，体积小 |
